========================================
  MX-jx 配置文件详细说明 v4.3.0
========================================

本项目的所有可配置项集中在 config/ 目录下，共 5 个 PHP 文件：

  config/api.php       ── API 解析线路列表
  config/platform.php  ── 平台规则 → 优先接口映射
  config/switch.php    ── 系统开关（总接口/缓存/M3U8/ZJK）
  config/admin.php     ── 后台配置（密码/端口/IP/会话/日志）
  config/noad.php      ── NoAd 去广告系统（关键词/代理/速率/算法/资源站）

所有配置文件的返回值都是 PHP 数组，加载方式：

  $apiConfig = require __DIR__ . '/config/api.php';

修改配置后立即生效，无需重启服务。

========================================
【1】config/api.php ── API 线路配置
========================================

文件路径：/workspace/config/api.php

返回格式：一维关联数组
  KEY   = 接口名称（任意字符串，唯一即可）
  VALUE = "接口URL|超时秒数"

  示例：
    'MyJson1-VIP' => "https://jx.jsonplayer.com/player/?url=|5"

完整字段说明：

  ┌─────────────────────────────────────────────────────────────┐
  │ 字段            说明                                         │
  ├─────────────────────────────────────────────────────────────┤
  │ 接口名称 (key)  任意字符串，唯一即可。会出现在响应的 strategy   │
  │                字段中，便于调试追踪哪条接口命中。             │
  │                例如 "MyJson1-VIP" / "Backup1"               │
  │                后台编辑时，key 不可重复，否则后者会覆盖前者。  │
  │                                                             │
  │ URL (value|前) 解析接口的完整 URL，必须包含 ?url= 占位符。    │
  │                系统会把用户传入的视频 URL 追加到此处后请求。  │
  │                示例：https://jx.jsonplayer.com/player/?url= │
  │                                                             │
  │ 超时 (value|后) 当前接口的单次请求超时时间，单位秒。          │
  │                主线路建议 3~8 秒，备用线路建议 8~15 秒。       │
  │                超时过短会导致正常响应被丢弃；过长则整体响应慢。│
  │                示例：|5 表示 5 秒超时                       │
  └─────────────────────────────────────────────────────────────┘

当前默认已提供的接口（可自由增减）：

  1. MyJson1-VIP  ~ MyJson6-VIP  ── 主线路（前 N 条作为"总接口"）
  2. MyJson7-VIP  ~ MyJson13-VIP ── 扩展线路
  3. Backup1 / Backup2           ── 备用线路

添加新接口示例：

  'NewAPI-VIP' => "https://你的解析接口/?url=|6",

删除某接口：
  直接删除对应行（或在后台可视化管理中点击删除）。

========================================
【2】config/platform.php ── 平台规则配置
========================================

文件路径：/workspace/config/platform.php

返回格式：一维关联数组
  KEY   = 平台备注（任意字符串，唯一即可）
  VALUE = "域名关键字|优先使用的API名称"

  示例：
    'qq' => 'v.qq.com|MyJson1-VIP'

工作原理：

  1. 用户请求的 URL 中包含 "v.qq.com"
  2. 系统匹配到平台规则 'qq'
  3. 优先请求 "MyJson1-VIP" 接口
  4. 若该接口失败，回退到并发请求其他接口

字段说明：

  ┌─────────────────────────────────────────────────────────────┐
  │ 字段                说明                                     │
  ├─────────────────────────────────────────────────────────────┤
  │ 域名关键字          URL 中应包含的字符串片段。系统使用         │
  │                    strpos() 做不区分大小写的简单匹配。       │
  │                    无需写完整域名，写关键字即可。             │
  │                    示例：v.qq.com / iqiyi.com / douyin.com │
  │                                                             │
  │ 优先 API 名称      必须与 config/api.php 中的 key 一致。     │
  │                    表示该平台优先使用哪条解析线路。           │
  │                    若此 API 失败，仍会并发请求其他 API。     │
  └─────────────────────────────────────────────────────────────┘

当前默认配置已支持的平台：
  腾讯视频 / 爱奇艺 / 虎扑 / 抖音 / 芒果 TV / 优酷
  PPTV / 乐视 / 搜狐 / 1905电影 / Bilibili / 西瓜视频 / Acfun

========================================
【5】config/noad.php ── NoAd 去广告系统
========================================

文件路径：/workspace/config/noad.php

这是内容最多的配置文件，分为 9 个逻辑区块：

  5.1 系统总开关（2 项）
  5.2 M3U8 去广告核心配置（3 项）
  5.3 缓存加速配置（3 项）
  5.4 多源自动匹配（4 项）
  5.5 广告关键词库（8 大分类，约 200+ 关键词）
  5.6 广告白名单（5 大类安全词）
  5.7 资源类型分类（7 种资源类型）
  5.8 资源站去广告（预置 dytt/西瓜/如意/爱奇艺/腾讯/优酷/芒果）
  5.9 自定义算法扩展（2 项）
  5.10 HTTP 代理 & 反封禁（6 项）
  5.11 速率限制（3 项）
  5.12 请求伪装（3 项）
  5.13 默认解析接口（4 条源）
  5.14 数据统计（3 项）
  5.15 SQLite 数据库路径（1 项）

5.1 系统总开关
  ─────────────────────────────────────────────────────────────
  noad_enabled        true   NoAd 去广告系统总开关。
                           false = 完全关闭 NoAd 功能，回退
                           到旧版并发解析模式。
  debug_mode          false  调试模式开关。开启后会输出更多调试
                           信息。生产环境建议关闭。

5.2 M3U8 去广告核心配置
  ─────────────────────────────────────────────────────────────
  enable_ad_filter     true   启用广告片段智能过滤。
                           系统会扫描 M3U8 播放列表的每个片段，
                           匹配广告关键词，命中的片段将被移除。
  enable_ts_proxy      true   TS 片段跨域代理。
                           当视频片段域名与播放器不一致时，
                           通过 noad_proxy.php 代理转发。
  ad_keyword_threshold 2     广告关键词命中阈值。
                           单个片段中命中的关键词数量超过此值
                           才判定为广告。默认 2，范围 1~5。
                           值越高越严格（误删少，但漏删多）。

5.3 缓存加速配置
  ─────────────────────────────────────────────────────────────
  cache_enabled        true   启用缓存加速。
  cache_ttl            1800   缓存有效期，单位秒（默认 30 分钟）。
  cache_dir            __DIR__."/../cache"  缓存文件存放目录。

5.4 多源自动匹配
  ─────────────────────────────────────────────────────────────
  enable_multi_source  true   启用多源自动匹配。
                           并发请求多个解析源，取最快有效响应。
  max_source_try       3     单视频最多尝试解析源数量。
  request_timeout      10    单源请求超时时间，单位秒。
  concurrent_limit     8     并发请求上限，防止瞬间请求过多。

5.5 广告关键词库（自动识别流媒体规则）
  ─────────────────────────────────────────────────────────────
  ad_keywords 数组，包含 8 大类约 200+ 关键词：

    A. 通用广告标识   ad, ads, advert, advertisement, sponsor
    B. 中文广告标识   广告, 插播, 片头广告, 推广, 贴片广告
    C. 视频平台广告   ad.qq.com, admob, 穿山甲, 广点通
    D. CDN/域名广告   adcdn, tracker, tongji, hm.baidu, 51.la
    E. 播放器/解析广告 tvod, txvideo_ad, playerad, preroll, postroll
    F. 推广/营销关键词 promo, banner, logo, skip, countdown
    G. 短片段广告     logo_, preroll_, _logo, teaser, bumper
    H. M3U8 片段路径   commercial, adsegment, adbreak, adblock

  匹配原理：
    1. 遍历 M3U8 播放列表的每个 #EXTINF 行及片段 URI
    2. 逐字段与 ad_keywords 做不区分大小写匹配
    3. 单个片段中命中的关键词数量 > ad_keyword_threshold
       则该片段被标记为广告并移除
    4. 白名单优先：若片段命中 whitelist_keywords 则保留

5.6 广告白名单（防止误伤）
  ─────────────────────────────────────────────────────────────
  whitelist_keywords 数组，包含 5 大类安全词：

    A. 正片/内容相关   main, content, video, 正片, trailer
    B. 分辨率/画质      hd, fhd, 4k, 1080p, 720p, h264
    C. 音频/字幕        audio, aac, subtitle, chn, eng
    D. 播放列表相关     playlist, index, master, stream, vod
    E. CDN/源站白名单   aliyun, tencent, bilibili, iqiyi, youku
       （常见视频平台 CDN 域名关键词，避免误删正片）

5.7 资源类型分类
  ─────────────────────────────────────────────────────────────
  resource_types 数组，共 7 种资源类型：

    ID  key        名称
    ───────────────────────────
    1   movie      🎬 电影资源
    2   tv         📺 电视剧集
    3   variety    🎤 综艺娱乐
    4   anime      🎭 动漫动画
    5   document   📚 纪录片
    6   sports     ⚽ 体育赛事
    7   short      📱 短视频

  type_id 用于区分解析源，可配置不同类型的视频使用
  不同的解析源列表。

5.8 资源站去广告（可在后台可视化管理）
  ─────────────────────────────────────────────────────────────
  resource_sites 数组，预置了 7 个常见资源站：

    电影天堂 (dytt)         patterns: dytt, 电影天堂, dy8
    西瓜资源 (xg)           patterns: xigua, xiguang, 西瓜, 360kan
    如意资源 (ry)           patterns: ruyi, 如意, ry.jx
    爱奇艺去广告 (iqiyi)     patterns: iqiyi, 爱奇艺
    腾讯视频去广告 (qq)     patterns: qq.com, 腾讯, v.qq
    优酷去广告 (youku)     patterns: youku, 优酷, yk
    芒果 TV (mgtv)         patterns: mgtv, 芒果, hunantv

  每个资源站包含字段：
    name        显示名称
    short_code  短代码
    base_url    资源站 URL
    patterns    URL 匹配关键词数组
    algorithms  对应清理算法数组（suanfa 系列）
    priority    优先级（数字越小越先匹配）
    enabled     是否启用

5.9 自定义算法扩展（v4.2 新增）
  ─────────────────────────────────────────────────────────────
  enable_custom_algorithms    true    是否启用 /algorithms 目录
                                    下的自定义算法。
  custom_algorithms_scope    'all'   默认作用域。
                                    可选：url / m3u8 / all
                                    url = 仅处理 URL 字符串
                                    m3u8 = 仅处理 M3U8 内容
                                    all = URL 和 M3U8 都处理

  编写自定义算法：复制 algorithms/custom_template.php，
  重命名并修改其中的 apply() 方法即可。

5.10 HTTP 代理 & 反封禁（v4.3 新增）
  ─────────────────────────────────────────────────────────────
  enable_proxy            false   是否启用 HTTP 代理池。
  proxies                []      多代理轮换池。
                                格式："http://ip:port" 或
                                "http://user:pass@ip:port"
                                示例："http://127.0.0.1:7890"
  proxy_failover          true    代理失败时自动退回到直连，
                                避免因某个代理挂掉整体不可用。
  proxy_random_user_agent true    随机化 User-Agent。
                                每次请求从 UA 池中随机选择一个
                                浏览器 UA，降低被识别为爬虫的概率。

5.11 速率限制（v4.3 新增）
  ─────────────────────────────────────────────────────────────
  enable_rate_limit           true   是否启用速率限制。
  rate_limit_min_interval_ms 350    同域名最小请求间隔，毫秒。
                                默认 350ms。设置越大越不容易封 IP，
                                但整体响应越慢。推荐 200~1000。
  rate_limit_burst_jitter_ms 250    额外随机抖动，毫秒。
                                让每次请求间隔不均匀（±抖动值），
                                更难被检测为自动化请求。推荐 100~500。
  rate_limit_concurrent_max  4      同一时刻最大并发请求数。
                                针对解析源，避免同时发太多请求。

5.12 请求伪装（v4.3 新增）
  ─────────────────────────────────────────────────────────────
  request_random_delay        true   顺序请求时每个请求之间加
                                    50~250ms 随机延迟。
  request_retry_on_failure    1     请求失败自动重试次数。
                                    0 = 关闭，默认 1。
  request_custom_headers      []    自定义请求头。
                                    键值对数组，可覆盖默认请求头。
                                    示例："X-Custom-Header" => "value"

5.13 默认去广告解析接口
  ─────────────────────────────────────────────────────────────
  default_sources 数组，可在后台可视化管理。
  每个源包含：name, url, timeout, type

  当前默认：
    Noad-主源   https://jx.playerjy.com/?url={url}  8s
    Noad-备源1  https://www.yemu.xyz/?url={url}     8s
    Noad-备源2  https://jx.xmflv.com/?url={url}     8s
    Noad-备源3  https://jx.aidouer.net/?url={url}  10s

  URL 中的 {url} 会被替换为实际视频 URL。

5.14 数据统计
  ─────────────────────────────────────────────────────────────
  stats_enabled          true   启用访问数据统计。
  stats_log_request      true   记录每次解析请求（IP/时间/来源/耗时）。
  stats_top_limit        20    热门统计展示条数。

5.15 SQLite 数据库路径
  ─────────────────────────────────────────────────────────────
  sqlite_path            __DIR__."/../cache/noad.db"
                       SQLite 数据库文件路径。
                       存储解析源、广告规则、访问日志、统计数据。
                       文件首次访问时自动创建，无需手动管理。

========================================
【ZJK.txt】 自定义接口文件（纯文本）
========================================

文件路径：/workspace/ZJK.txt

格式：每行一个接口，以 "接口URL|超时秒数" 形式写入
注释行以 # 开头，将被忽略

示例内容：

  # 这是注释行
  https://jx.example.com/player/?url=|6
  https://jx2.example.com/?url=|8

特点：
  - 改动即时生效，无需重启
  - 方便批量管理（可从其他系统导入）
  - 与 config/api.php 中的接口一起参与并发请求

========================================
【配置修改的常见场景】
========================================

场景 1：添加新的解析接口

  方法 A（推荐，可视化）：登录后台 → API 线路 → 添加
  方法 B（手动）：编辑 config/api.php，添加一行：
    'NewAPI' => 'https://接口/?url=|6',

场景 2：提高响应速度（减少并发）

  编辑 config/switch.php：
    'global_api_count' => 3,     // 减少到 3 条
    'global_api_timeout' => 5,   // 超时缩短到 5 秒

场景 3：仅使用自己的接口（不使用默认解析接口）

  编辑 config/api.php，删除默认接口，添加自己的接口。
  或编辑 ZJK.txt，添加自定义接口。

场景 4：添加视频平台规则

  编辑 config/platform.php，添加一行：
    'NewPlatform' => '域名关键字|优先API名称',

  例如：'Bilibili' => 'bilibili.com|MyJson5-VIP',

场景 5：启用代理防止 IP 被封

  编辑 config/noad.php：
    'enable_proxy' => true,
    'proxies' => [
        "http://127.0.0.1:7890",
        "http://127.0.0.1:10809",
    ],

场景 6：修改后台密码

  方法 A（推荐）：登录后台 → 修改密码页面，输入新旧密码提交
  方法 B（手动）：echo md5('新密码');  // 得到哈希值
                    替换 config/admin.php 中 admin_password 的值
