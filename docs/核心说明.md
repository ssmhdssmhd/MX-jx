========================================
【核心数据流程详解】
========================================

流程 1：常规视频解析（用户请求 index.php?url=...）
────────────────────────────────────────────────

  阶段 1 ── 入口校验（index.php）
  ├─ 校验 $_GET['url'] 非空
  ├─ 校验 URL 格式（filter_var FILTER_VALIDATE_URL）
  └─ 校验协议（仅允许 http/https，防止 SSRF）

  阶段 2 ── M3U8 直链检测
  ├─ URL 包含 ".m3u8" 或 "m3u8" 关键字？
  ├─ 是 → 调用 M3U8Handler::generateM3U8Response()
  │     直接返回 JSON 响应，不经过解析 API
  │     响应字段：code=200, msg=URL, url=URL,
  │               strategy="M3U8-Direct-Output"
  └─ 否 → 进入下一阶段

  阶段 3 ── 模式判定（mode 参数）
  ├─ mode=noad（且 noad_enabled=true）→ 进入 NoAd 流程
  ├─ mode=legacy → 进入旧版并发流程
  └─ mode=auto（默认）→ 优先 NoAd，失败回退 legacy

  阶段 4 ── NoAd 解析（mode=noad 或 auto）
  ├─ 加载 config/noad.php
  ├─ 实例化 NoAdParser
  ├─ NoAdParser::parse($url, $type)
  │   ├─ 步骤 A：匹配资源站规则（suanfa 系列算法）
  │   │     对照 config/noad.php 中 resource_sites
  │   │     若 URL 命中 patterns → 应用对应 algorithms
  │   ├─ 步骤 B：匹配解析源（多源并发请求）
  │   │     并发请求 default_sources 中的解析接口
  │   │     取第一个返回有效 m3u8 的结果
  │   ├─ 步骤 C：M3U8 广告片段过滤
  │   │     解析 m3u8 文件
  │   │     遍历每个 #EXTINF + 片段 URI
  │   │     与 ad_keywords 匹配
  │   │     与 whitelist_keywords 匹配（白名单优先）
  │   │     命中数 > ad_keyword_threshold 则移除
  │   ├─ 步骤 D：自定义算法扩展
  │   │     扫描 algorithms/*.php
  │   │     按 priority 排序
  │   │     根据 scope（url/m3u8/all）依次调用 apply()
  │   └─ 步骤 E：记录统计
  │         写入 SQLite access_log 表
  │         更新 stats_daily 计数
  └─ 返回 JSON 响应

  阶段 5 ── 旧版并发解析（mode=legacy 或 NoAd 失败回退）
  ├─ 加载 config/api.php, platform.php, switch.php
  ├─ 实例化 Strategy
  │   └─ Strategy::getBestApiForUrl($url)
  │       ├─ 遍历 platform 规则，匹配平台
  │       ├─ 命中则返回对应优先 API + 其他 API
  │       └─ 未命中则返回所有 API（global_api_count 条）
  ├─ 实例化 Requester
  │   └─ Requester::concurrentRequest($url, $apiList)
  │       ├─ 初始化 curl_multi
  │       ├─ 为每条 API 创建 cURL 句柄
  │       ├─ 设置 CURLOPT_TIMEOUT, CURLOPT_USERAGENT
  │       ├─ 设置代理（如启用）
  │       ├─ 并发执行 curl_multi_exec
  │       ├─ 收集 HTTP 200 + 非空响应
  │       ├─ 按优先级排序（优先 API > 总接口 > 其他）
  │       └─ 统一 JSON 格式返回
  └─ 返回 JSON 响应

========================================
【Strategy 策略模块设计】
========================================

类文件：core/strategy.php

核心职责：根据视频 URL 决定使用哪组 API 线路解析。

设计要点：
  1. 两级决策
     ├─ 第一级：平台规则匹配（精准匹配，最高优先级）
     └─ 第二级：总接口回退（无匹配时使用前 N 条 API）

  2. 平台规则格式
     '平台名称' => '域名关键字|优先API名称'

  3. 策略输出
     给 Requester 提供：
     ├─ priorityApi  ── 优先接口（平台匹配命中的）
     ├─ globalApis   ── 总接口（前 global_api_count 条）
     └─ otherApis    ── 其他接口（所有线路的剩余部分）

  4. 回退机制
     若优先接口全部失败，自动回退并发请求其他接口。

核心方法：
  getBestApiForUrl($url)        → 确定 API 列表
  matchPlatform($url)           → 匹配平台规则
  parseApiConfig($apiConfig)    → 解析 api.php 配置

========================================
【Requester 请求模块设计】
========================================

类文件：core/requester.php

核心职责：并发请求多个 API，收集并返回最优结果。

技术要点：
  1. curl_multi 并发
     - 初始化 curl_multi_init
     - 为每条 API 添加 curl 句柄
     - curl_multi_exec 轮询执行
     - curl_multi_getcontent 获取响应内容

  2. 响应筛选
     - HTTP 状态码 = 200
     - 响应体非空
     - 响应体为合法 JSON（可选）

  3. 结果排序
     - 优先使用平台匹配的 API 响应
     - 其次使用总接口中最快的响应
     - 最后使用其他接口中最快的响应

  4. 代理支持（v4.3 新增）
     - 从 config/noad.php 读取 proxies 数组
     - 随机选择一个代理
     - 设置 CURLOPT_PROXY
     - 失败时自动回退到直连（proxy_failover=true）

  5. 速率限制（v4.3 新增）
     - 同域名最小请求间隔：rate_limit_min_interval_ms
     - 随机抖动：rate_limit_burst_jitter_ms
     - 最大并发：rate_limit_concurrent_max
     - 通过 usleep() 实现间隔控制

核心方法：
  concurrentRequest($url, $apiList)   → 主入口，并发请求
  formatResponse($raw, $apiName)       → 统一响应格式

========================================
【M3U8Handler 处理器设计】
========================================

类文件：handlers/M3U8Handler.php

核心职责：快速识别 m3u8 直链并返回响应，跳过解析流程。

工作原理：
  1. 检查 URL 关键字
     - 包含 ".m3u8" 扩展名
     - 或路径/查询参数包含 "m3u8"

  2. 若命中 → 直接返回解析成功响应
     - code = 200
     - msg = url = 原始 URL
     - strategy = "M3U8-Direct-Output"

  3. 若未命中 → 返回 false，进入常规解析流程

意义：
  - 对已有 m3u8 链接的请求，无需经过解析 API
  - 响应时间从数秒降低到 <10ms
  - 减少对解析接口的依赖和请求量

核心方法：
  isM3U8Url($url)                    → 检测是否为 m3u8 URL
  generateM3U8Response($url, $raw)   → 生成标准响应

【NoAdParser 去广告核心设计】
========================================

类文件：core/NoAdParser.php

核心职责：M3U8 广告片段过滤 + 资源站清洗 + 自定义算法。

处理流水线：

  输入视频 URL
       │
       ▼
  ┌────────────────┐
  │  资源站匹配      │  ← config/noad.php resource_sites
  │  (suanfa 系列)   │    按 patterns 匹配，应用对应算法
  └────────┬───────┘
           │
           ▼
  ┌────────────────┐
  │  解析源并发请求  │  ← default_sources + SQLite parse_sources
  │  (多源自动匹配)  │    curl_multi 并发，取最快有效 m3u8
  └────────┬───────┘
           │
           ▼
  ┌────────────────┐
  │  M3U8 解析     │  ← 解析 m3u8 文本 → 片段列表
  │  (EXT-X-* 标签)│    识别 #EXTINF / #EXT-X-STREAM-INF
  └────────┬───────┘
           │
           ▼
  ┌────────────────┐
  │  广告关键词扫描  │  ← ad_keywords + whitelist_keywords
  │  (逐片段扫描)    │    命中数 > threshold → 标记为广告
  └────────┬───────┘
           │
           ▼
  ┌────────────────┐
  │  自定义算法链   │  ← algorithms/ 目录
  │  (可插拔扩展)    │    AbstractAlgorithm 子类
  │                  │    AlgorithmRegistry 扫描/排序/应用
  │                  │    按 scope(url/m3u8/all) 调用 apply()
  └────────┬───────┘
           │
           ▼
  ┌────────────────┐
  │  生成新 M3U8   │  ← 仅保留非广告片段
  │  (或返回解析URL)│    重新写入 EXT-X-* 头
  └────────┬───────┘
           │
           ▼
  ┌────────────────┐
  │  记录统计      │  ← SQLite access_log + stats_daily
  │  (访问/广告/耗时)│   写入解析时间/IP/来源/耗时/广告移除数
  └────────┬───────┘
           │
           ▼
  返回 JSON 响应

核心数据结构：

  NoAdParser::parse() 返回数组：
  ┌──────────────────────────────────────────┐
  │ 字段              类型      说明           │
  ├──────────────────────────────────────────┤
  │ code             int       200 = 成功     │
  │ msg              string    提示消息        │
  │ url              string    解析后播放地址 │
  │ original_url     string    原始 URL        │
  │ ads_removed      int       移除的广告片段数│
  │ total_segments   int       总片段数        │
  │ parser_source    string    使用的解析源     │
  │ processing_time  float     处理耗时（秒）  │
  │ custom_algo      array     自定义算法结果   │
  │ m3u8_content     string    清洗后的 m3u8  │
  │ strategy         string    策略标记         │
  └──────────────────────────────────────────┘

核心方法：
  parse($url, $type='')                → 主入口
  parseM3U8($content, $baseUrl)        → M3U8 扫描广告
  cleanByResourceSite($url)            → 资源站规则匹配
  applyCustomAlgorithms($input, $ctx)  → 自定义算法链
  listCustomAlgorithms()                → 列出所有已加载算法
  setCustomAlgorithmEnabled($name, $v) → 开关算法
  reloadCustomAlgorithms()              → 重新扫描目录

【SmartCache 缓存模块设计】
========================================

类文件：core/cache.php

核心职责：文件系统缓存 + TTL 自动过期。

设计要点：
  1. 文件存储
     - cache/ 目录下，按 key 生成文件名
     - 文件名：md5(key) + ".cache"
     - 文件内容：serialize([value, expireAt])

  2. TTL 自动过期
     - 读取时检查文件 mtime + ttl 是否 > time()
     - 过期文件自动删除
     - 懒删除，无需定时任务

  3. 键值结构
     key 命名约定：<domain>:<identifier>
     示例：noad:url:https://v.qq.com/...

核心方法：
  get($key)                       → 读缓存（过期返回 false）
  set($key, $value, $ttl=1800)    → 写缓存
  delete($key)                    → 删除单个键
  clearAll()                      → 清空所有缓存

========================================
【自定义算法扩展体系（v4.2 新增）】
========================================

目录：algorithms/

设计模式：策略模式（Strategy Pattern）+ 注册表模式

类层次结构：

  AbstractAlgorithm（抽象基类）
    ├─ name(): string          ── 算法名称
    ├─ priority(): int         ── 执行优先级（大的先执行）
    ├─ scope(): string         ── 作用域（url/m3u8/all）
    ├─ matchPatterns(): array  ── URL 匹配模式（空=始终执行）
    ├─ shouldRun($ctx): bool   ── 是否应当执行
    └─ apply($input, $ctx): mixed  ── 算法主体

  AlgorithmRegistry（注册表）
    ├─ scanDirectory($dir)     ── 扫描目录中的算法
    ├─ register($algo)         ── 注册单个算法实例
    ├─ applyByScope($scope, $input, $ctx)  ── 按作用域应用
    ├─ listAll()               ── 列出所有已注册算法
    ├─ toggleEnabled($name)    ── 切换启用状态
    └─ reload()                ── 重新扫描目录

  ExampleAlgorithm（示例算法，多个）
    1. example_01_url_tracker_strip.php  ── 去除 URL 跟踪参数
    2. example_02_ad_keyword_filter.php  ── 广告关键词过滤
    3. example_03_domain_rewrite.php      ── 广告域名重写
    4. example_04_m3u8_segment_cleaner.php ── M3U8 片段清洗
    5. example_05_protocol_normalize.php  ── 协议规范化

  CustomTemplate（开发模板）
    - algorithms/custom_template.php 可复制改名后编写

算法编写流程：

  1. 复制 algorithms/custom_template.php → 新文件 my_algo.php
  2. 定义类名：class MyAlgo extends AbstractAlgorithm
  3. 实现 name() / priority() / scope() / matchPatterns()
  4. 实现 apply($input, $context):
     - $input 是当前处理的字符串（URL 或 M3U8 内容）
     - $context 是上下文数组（含 url, type, config 等）
     - 返回处理后的字符串，或 ['modified' => ..., 'removed' => n]
  5. 保存文件，算法注册表会自动扫描加载

算法启用/禁用：
  - 通过后台「🧩 自定义算法」Tab 可视化管理
  - 通过 API：noad_proxy.php?mode=algorithms&a=toggle&name=XXX
  - 状态保存在 SQLite 或内存中

【请求/响应格式规范】
========================================

对外 API 格式（index.php 返回）：

  {
    "code": 200,           // 200 = 成功，其他 = 失败
    "msg":  "https://...", // 消息（统一显示时与 url 相同）
    "url":  "https://...", // 解析后的播放地址
    "type": "m3u8",        // 资源类型
    "MX":   "...",         // MX 提示信息
    "strategy": "MyJson1-VIP-Global-Fastest", // 命中策略
    "developer": "MX-射手沫蝴蝶",
    "contact":  "QQ: 2094332348",
    "timestamp": 1718768000,
    "switch_status": {
      "global_api_enabled": true,
      "global_api_count": 6,
      "zjk_apis_enabled": true,
      "zjk_file_exists": true,
      "unified_display_enabled": true,
      "m3u8_direct_enabled": true
    }
  }

策略字段（strategy）含义：

  <API名称>-<类型>
  │         │
  │         └─ Priority[平台名]  = 平台优先接口命中
  │             Global-Fastest    = 总接口最快
  │             Fastest           = 普通接口最快
  │             Sequential        = 顺序重试成功
  │
  └─ M3U8-Direct-Output           = M3U8 直链（跳过解析）

错误响应示例：

  {
    "code": 400,
    "msg":  "URL参数不能为空",
    "developer": "MX-射手沫蝴蝶",
    "contact": "QQ: 2094332348",
    "timestamp": 1718768000
  }

性能优化建议：

  1. 减少并发数：global_api_count 从 6 降到 3
  2. 启用缓存：cache_enabled = true, cache_ttl = 3600
  3. 启用代理池：proxies 填入可用代理
  4. 启用自定义算法：对高频场景编写专用算法
  5. SQLite 数据库放在 SSD 分区（如有）

安全加固建议：

  1. 修改默认密码 admin123（首次登录必做）
  2. 启用 IP 白名单（仅允许你的服务器或办公 IP）
  3. 启用端口限制，并在 Nginx 中仅允许特定端口访问后台
  4. 修改 admin_path 为不易猜测的文件名
  5. 设置会话有效期 < 1 小时
  6. 对外暴露 index.php 时，建议增加 Referer 检查
