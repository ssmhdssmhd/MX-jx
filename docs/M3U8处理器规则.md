========================================
  M3U8 处理器规则与解析规范 v4.3.0
========================================

本文档详细说明 M3U8 格式、项目中的 M3U8 处理流程、
广告识别规则、以及如何扩展自定义的 M3U8 处理逻辑。

========================================
【M3U8 基础知识】
========================================

M3U8 是 HLS (HTTP Live Streaming) 的播放列表文件格式，
本质是 UTF-8 编码的纯文本，包含一系列媒体片段 URL 和
相关标签（EXT-X-*）。

基本结构示例：

  #EXTM3U
  #EXT-X-VERSION: 3
  #EXT-X-TARGETDURATION: 10
  #EXT-X-MEDIA-SEQUENCE: 0
  #EXTINF: 9.009,
  segment00001.ts
  #EXTINF: 10.0,
  segment00002.ts
  #EXTINF: 9.5,
  segment00003.ts
  #EXT-X-ENDLIST

常用标签说明：

  ┌────────────────────────────────────────────────────────┐
  │ 标签                 含义                                │
  ├────────────────────────────────────────────────────────┤
  │ #EXTM3U            文件头（必须在第一行）                 │
  │ #EXT-X-VERSION:n   HLS 协议版本                         │
  │ #EXT-X-TARGETDURATION:n  最大片段时长（秒）              │
  │ #EXT-X-MEDIA-SEQUENCE:n  第一个片段的序列号              │
  │ #EXTINF:duration,  下一个片段的时长（秒）                │
  │ #EXT-X-DISCONTINUITY  前后片段编码不同（用于广告插入）  │
  │ #EXT-X-KEY:METHOD=X  加密信息（DRM）                     │
  │ #EXT-X-STREAM-INF:  多码率适配（主播放列表）            │
  │ #EXT-X-ENDLIST     播放列表结束（直播通常没有此标签）   │
  └────────────────────────────────────────────────────────┘

========================================
【M3U8 处理流程（本项目）】
========================================

流程 1：M3U8Handler 快速通道（直链识别）
──────────────────────────────────────────

  入口：index.php 收到 ?url=xxx 请求
  │
  └─ 调用 M3U8Handler::isM3U8Url($url)
      │
      ├─ 检查 URL 是否包含 ".m3u8" 扩展名
      ├─ 检查 URL 路径是否包含 "m3u8" 关键字
      └─ 检查 URL 查询参数是否包含 "m3u8" 关键字
      │
      └─ 命中 → 直接返回 JSON 响应
            {
              code: 200,
              url: "<原始URL>",
              msg: "<原始URL>",
              strategy: "M3U8-Direct-Output"
            }

  意义：对已是 m3u8 的链接，跳过解析 API，<10ms 返回

流程 2：NoAdParser M3U8 广告片段过滤
──────────────────────────────────────────

  入口：NoAdParser::parseM3U8($content, $baseUrl)

  步骤 1：解析文本为结构化数据
    ├─ 逐行扫描
    ├─ 识别 #EXTINF 行及其下一行的片段 URI
    ├─ 识别 #EXT-X-* 头信息
    └─ 构建 segments[] 数组（每个元素含 duration, uri, tags）

  步骤 2：对每个片段进行广告识别
    对 segment i:
    ├─ 合并 #EXTINF 行 + URI 到字符串 text
    ├─ 初始化 hitCount = 0
    ├─
    ├─ 白名单检查（优先，避免误删正片）：
    │   ├─ 遍历 config/noad.php whitelist_keywords
    │   └─ 命中任一关键词 → 标记为 SAFE，跳过后续检查
    │
    ├─ 广告关键词检查：
    │   ├─ 遍历 config/noad.php ad_keywords
    │   └─ 命中每个关键词 → hitCount++
    │
    ├─ 广告域名检查：
    │   ├─ 解析 URI 的 host 部分
    │   └─ 与 ad_keywords 中的域名类关键词比对
    │
    └─ 判定：
        ├─ hitCount > ad_keyword_threshold → 标记为 AD
        ├─ 片段时长 < 1 秒且含 "ad" → 标记为 AD
        └─ 其他 → 保留为 CONTENT

  步骤 3：生成清洗后的 M3U8
    ├─ 保留原始 #EXTM3U、#EXT-X-VERSION、#EXT-X-TARGETDURATION 等头
    ├─ 保留所有 hitCount <= threshold 的片段
    ├─ 更新 #EXT-X-MEDIA-SEQUENCE（如有移除开头片段）
    └─ 保留 / 生成新的 #EXT-X-ENDLIST

  步骤 4：返回结果
    {
      original_segments: 150,
      ads_removed: 12,
      m3u8_content: "...",
      ad_keywords_hit: ["ad_", "promo", "广告"]
    }

流程 3：NoAdParser M3U8 资源站匹配清洗
──────────────────────────────────────────

  入口：NoAdParser::cleanByResourceSite($url)

  步骤 1：匹配资源站
    遍历 config/noad.php resource_sites
    对每个资源站：
      ├─ 获取 patterns[]（匹配关键字数组）
      ├─ 检查 URL 是否包含任一 pattern
      └─ 命中 → 记录 resource_name 和对应 algorithms[]

  步骤 2：应用资源站专用算法
    对命中的 algorithms 列表，依次执行：
      suanfa_small($url)       // 小型资源站清洗
      suanfa3($url)            // 特殊参数处理
      suanfa_dyt($url)        // dytt 专用
      ... 等

  步骤 3：返回结果
    {
      matched_site: "dytt",
      algorithms_applied: ["suanfa_small", "suanfa3"],
      ad_tokens_removed: 3,
      final_url: "https://..."
    }

========================================
【广告关键词识别规则详解】
========================================

默认 ad_keywords 库分类（共约 200+ 关键词）：

  分类 A：通用广告标识（约 30 个）
    ad, ads, advert, advertisement, pre-roll, mid-roll,
    post-roll, adbreak, adslot, adzone, sponsor, promotional,
    promo, 广告, 推广, 贴片, 暂停广告, 片头广告, 片中广告,
    片尾广告, 角标广告, 浮动广告

  分类 B：CDN / 域名广告相关（约 40 个）
    adcdn, adpush, adp, adstatic, adv., -adv, /adv/, ads/,
    click, clicktrack, clickid, clickTAG, tracker, tracking,
    analytics, tongji, cnzz, hm.baidu, 51.la, pixel, beacon

  分类 C：播放器 / 解析广告（约 30 个）
    tvod, txvideo_ad, aliyun_ad, qiyi_ad, youku_ad,
    playerad, player_ad, preroll, postroll, midroll, videoad,
    video_ad, clipad, clip_ad, mp4ad, ad.mp4, ad.m3u8, ad_

  分类 D：短片段广告（含时长辅助判断）（约 30 个）
    logo_, _logo, logo., corner_ad, preroll_, _preroll,
    postroll_, _postroll, short_, _short, teaser, bumper,
    intro_ad, outro_ad, 片头, 片尾, 角标, 贴片

  分类 E：其他（约 30 个）
    commercial, commercials, ad_content, ad_clip, adsegment,
    ad_seg, adv_seg, adpiece, breakpoint, ad_block, noncontent,
    non_content, placeholder_ad, overlay, overlay_ad, watermark_ad

识别优先级逻辑：

  1. 白名单优先（whitelist_keywords）：
     若片段 URI 或 #EXTINF 行包含 main, content, video,
     正片, hd, fhd, 4k, 1080p, 720p, bilibili, iqiyi,
     youku 等白名单关键词 → 强制保留，不进行广告检查。

  2. 广告关键词计数：
     其他片段，对每个 ad_keywords 做大小写不敏感匹配，
     命中次数计入 hitCount。

  3. 阈值判定：
     hitCount > ad_keyword_threshold（默认 2）→ 判定为广告
     可通过 config/noad.php 修改 threshold 调整严格度。

  4. 时长辅助判定：
     片段 duration < 1.0 秒且含 "ad/广告/promo/logo"
     → 强化为广告（额外 +2 hitCount）。

  5. #EXT-X-DISCONTINUITY 处理：
     遇到 DISCONTINUITY 标签，通常意味着编码切换或广告插入。
     - 检查后续 1~2 个片段是否为广告
     - 是广告则连同 DISCONTINUITY 一起移除
     - 非广告则保留 DISCONTINUITY 以保证播放连续性

【自定义算法（v4.2 新增）与 M3U8 结合】
========================================

位置：algorithms/ 目录

编写对 M3U8 生效的自定义算法（推荐模板）：

  <?php
  // 文件：algorithms/my_m3u8_cleaner.php

  require_once __DIR__ . '/AbstractAlgorithm.php';

  class MyM3u8Cleaner extends AbstractAlgorithm
  {
      public function name(): string { return 'my_m3u8_cleaner'; }
      public function priority(): int { return 50; }   // 数字越大越先执行
      public function scope(): string { return 'm3u8'; }  // 仅对 M3U8 内容执行
      public function matchPatterns(): array { return []; }  // 空数组表示对所有 URL 都执行

      public function apply($input, $context = [])
      {
          // $input 是 m3u8 文件文本
          // $context 是数组，含 url, type, config, noad_parser 等
          $lines = explode("\n", $input);
          $outputLines = [];
          $removed = 0;

          foreach ($lines as $line) {
              // 你的处理逻辑：
              // 例如：去除含 "ad_video" 且时长 < 3 秒的片段
              if (stripos($line, 'ad_video') !== false) {
                  $removed++;
                  continue; // 跳过这一行（及下一行 URI）
              }
              $outputLines[] = $line;
          }

          // 返回处理后的文本
          return [
              'modified' => implode("\n", $outputLines),
              'removed'  => $removed,
          ];
      }
  }

  // 注册到注册表（文件末尾）
  AlgorithmRegistry::register(new MyM3u8Cleaner());

算法作用域（scope）说明：

  ┌────────────────────────────────────────────────────────┐
  │ scope 值      调用时机              传入 $input 类型    │
  ├────────────────────────────────────────────────────────┤
  │ 'url'        URL 阶段              字符串（视频 URL）  │
  │              NoAdParser::parse 开始时                   │
  │              常用于：移除跟踪参数、域名重写、协议统一  │
  │                                                          │
  │ 'm3u8'       M3U8 内容阶段         字符串（m3u8 全文） │
  │              NoAdParser::parseM3U8 广告过滤后调用       │
  │              常用于：额外规则清洗、片段去重、格式规整  │
  │                                                          │
  │ 'all'        URL + M3U8 两个阶段都执行                 │
  │              需要在 apply() 内部判断 $input 是 URL 还是 m3u8│
  │              常用于：综合处理（域名替换 + 片段清洗）     │
  └────────────────────────────────────────────────────────┘

算法注册表（AlgorithmRegistry）调用链：

  NoAdParser::parse($url)
    │
    ├─ applyCustomAlgorithms('url', $url, $context)
    │   └─ AlgorithmRegistry::applyByScope('url', ...)
    │         └─ 扫描所有 scope='url' 或 'all' 的算法实例
    │             按 priority 降序调用 apply()
    │
    ├─ 解析源并发请求
    ├─ M3U8 内容获取
    │
    ├─ 广告关键词过滤（默认规则）
    │
    ├─ applyCustomAlgorithms('m3u8', $m3u8Content, $context)
    │   └─ AlgorithmRegistry::applyByScope('m3u8', ...)
    │         └─ 扫描所有 scope='m3u8' 或 'all' 的算法实例
    │             按 priority 降序调用 apply()
    │
    └─ 记录统计 + 返回 JSON

========================================
【noad_proxy.php 代理转发机制】
========================================

入口：noad_proxy.php?mode=<mode>&url=<视频URL或m3u8URL>

mode 取值：

  ┌────────────────────────────────────────────────────────┐
  │ mode        功能              典型使用场景               │
  ├────────────────────────────────────────────────────────┤
  │ 'm3u8'     M3U8 清洗代理      播放器直接请求清洗后的    │
  │                              m3u8（无缝屏蔽广告）       │
  │                              URL: noad_proxy.php?mode=m3u8&url=xxx.m3u8│
  │                              返回: Content-Type: application/x-mpegURL │
  │                                      清洗后的 m3u8 文本                  │
  │                                                           │
  │ 'ts'       TS 片段跨域代理    解决 m3u8 内的 ts 片段     │
  │                              跨域无法播放问题             │
  │                              URL: noad_proxy.php?mode=ts&url=xxx.ts     │
  │                              返回: Content-Type: video/MP2T             │
  │                                      二进制 TS 数据                       │
  │                                                           │
  │ 'api'      JSON API           供第三方系统调用（返回解析结果）│
  │                              URL: noad_proxy.php?mode=api&url=xxx        │
  │                              返回: JSON，结构同 index.php                 │
  │                                                           │
  │ 'algorithms'  算法管理 API    供后台或第三方管理自定义算法│
  │   sub-action:  a=list        GET  列出所有算法及状态     │
  │                a=toggle      POST 切换某个算法启用/禁用  │
  │                a=reload      POST 重新扫描 algorithms/ 目录│
  │                a=test        POST 用测试 URL 执行算法链  │
  │                              返回: JSON                   │
  └────────────────────────────────────────────────────────┘

'mode=m3u8' 详细流程：

  1. 参数检查：$_GET['url'] 非空 + 合法 URL + http/https 协议
  2. 读取远程 m3u8 文件：curl / file_get_contents（遵守代理 + 速率限制）
  3. NoAdParser::parseM3U8() 过滤广告片段
  4. 若启用 custom algorithms → applyByScope('m3u8', ...) 进一步处理
  5. 重写片段 URI（如需跨域，改写为 noad_proxy.php?mode=ts&url=...）
  6. 设置响应头：Content-Type: application/x-mpegURL
  7. 设置缓存：Cache-Control: max-age=3600（视情况调整）
  8. 输出清洗后的 m3u8 文本

'mode=ts' 详细流程：

  1. 参数检查：$_GET['url'] 非空
  2. 读取远程 TS 文件（二进制）：
     - 使用 cURL，设置 CURLOPT_BINARYTRANSFER
     - 使用代理（如启用）
     - 遵守速率限制（如启用）
  3. 透传响应头：Content-Type（video/MP2T 或 application/octet-stream）
  4. 设置 CORS 头：Access-Control-Allow-Origin: *
  5. 输出二进制 TS 数据

'mode=api' 详细流程：

  1. 同 index.php 流程，但强制走 NoAd 解析路径
  2. 返回标准 JSON（含 code/url/strategy/ads_removed 等）

'mode=algorithms' 详细流程：

  a=list：
    - 实例化 AlgorithmRegistry
    - 调用 scanDirectory(__DIR__ . '/algorithms')
    - listAll() 获取算法数组
    - JSON 格式返回（name, priority, scope, matchPatterns, enabled）

  a=toggle（POST 参数 name=算法名称）：
    - AlgorithmRegistry::toggleEnabled($name)
    - 返回更新后的列表 JSON

  a=reload（POST）：
    - AlgorithmRegistry::reload()
    - 重新扫描目录并重新注册
    - 对开发调试非常有用（修改算法文件后无需重启）

  a=test（POST 参数 url=测试URL）：
    - 下载测试 URL 的 m3u8 或 传入文本
    - 依次调用所有 enabled 算法
    - 返回每个算法的处理结果 + 耗时 + 移除片段数

【代理与速率限制配置（v4.3 新增）】
========================================

位置：config/noad.php 中 enable_proxy / proxies 等参数

代理池轮换原理：

  当 enable_proxy = true 且 proxies 数组非空时：
    1. 每次请求从 proxies 中随机选择一个代理
    2. 设置 CURLOPT_PROXY 为选中的代理
    3. 设置 CURLOPT_HTTPPROXYTUNNEL = true（支持 HTTPS）
    4. 设置 CURLOPT_TIMEOUT（如启用 request_timeout）
    5. 执行请求
    6. 若 proxy_failover = true 且请求失败：
       - 退回到无代理直连重试
       - 记录本次代理失败事件

代理池配置示例：

  "enable_proxy" => true,
  "proxies" => [
      "http://127.0.0.1:7890",                 // 本地代理
      "http://user:pass@proxy.example.com:8080", // 带认证的代理
      "http://10.0.0.5:3128",                   // 内网代理
  ],
  "proxy_failover" => true,              // 代理失败自动回退直连

随机 User-Agent：

  proxy_random_user_agent = true 时：
    每次请求从以下 UA 池中随机选择一个填入请求头：
    - "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36..."
    - "Mozilla/5.0 (iPhone; CPU iPhone OS 15_0 like Mac OS X)..."
    - "Mozilla/5.0 (Linux; Android 12; Mobile)..."
    - "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36..."
    - "Mozilla/5.0 (Macintosh; Intel Mac OS X 12_5)..."

  目的：降低被识别为爬虫 / 自动化请求的概率。

速率限制原理：

  对同一目标域名（提取 host）维护请求时间队列：

    变量：
      rate_limit_min_interval_ms  = 350   // 最小间隔 350ms
      rate_limit_burst_jitter_ms  = 250   // 抖动 ±250ms
      rate_limit_concurrent_max   = 4     // 最大并发 4

    每次请求前：
      1. 计算上次请求距现在的时间
      2. 若 time_since_last_request < min_interval_ms + random_jitter
         → 执行 usleep(remaining_ms * 1000) 等待
      3. 检查当前并发数
         → 若 concurrent >= max，等待其他请求完成
      4. 更新上次请求时间戳

    注意：速率限制仅影响同一 PHP 进程内部，不适用于多进程
    （如 Nginx + PHP-FPM 多进程环境下每个进程独立计数）

随机延迟（request_random_delay）：

  当 request_random_delay = true 时：
    每次顺序请求前随机等待 50~250ms
    配合 random UA，进一步模拟真实用户行为。

自动重试（request_retry_on_failure）：

  当 request_retry_on_failure > 0 时：
    若首次请求失败（HTTP != 200，或超时），
    自动重试 N 次，每次更换代理（如启用代理池）。

     action = ajax_parse_m3u8

     - 统计信息显示在底部

【扩展建议：高级 M3U8 处理技巧】
========================================

技巧 1：根据域名使用专用解析源

  在 config/noad.php 的 default_sources 中为不同视频平台
  配置专用解析接口，提高解析成功率和去广告效果。

  "default_sources" => [
      // 通用
      ["name" => "NoAd-主源", "url" => "https://jx.playerjy.com/?url={url}", "timeout" => 8, "type" => 1],
      // 爱奇艺专用（示例）
      ["name" => "NoAd-iqiyi", "url" => "https://jx.iqiyi.example.com/?url={url}", "timeout" => 10, "type" => 1],
  ]

  type 字段对应资源类型：
    1 = 电影   2 = 电视剧   3 = 综艺
    4 = 动漫   5 = 纪录片   6 = 体育   7 = 短视频

技巧 2：对某些平台禁用直链检测

  在 config/switch.php 中 enable_m3u8_direct = false
  即可强制对所有 URL 走解析 API 路径（用于需要去广告的平台）。

技巧 3：为 m3u8 中的 ts 片段启用跨域代理

  当启用 noad_proxy.php?mode=ts 代理时：
    NoAdParser::parse() 返回的 url 会自动重写为
    noad_proxy.php?mode=ts&url=<原始 ts 片段 URL>
    这样播放器请求时会通过服务器转发，解决跨域问题。

技巧 4：使用算法注册表扩展清洗规则

  参考 algorithms/example_04_m3u8_segment_cleaner.php，
  编写自定义算法，对特定来源的 m3u8 做深度清洗。
  无需修改核心代码，即插即用。

========================================
【常见问题排查】
========================================

问题 1：清洗后的 m3u8 无法播放

  排查步骤：
    a. 打开原始 m3u8，确认能正常播放
       └─ 不能播放 → 源文件损坏，与本项目无关

    b. 对比清洗前后的 #EXT-X-MEDIA-SEQUENCE
       └─ 若移除了前面的片段，需更新 SEQUENCE 号

    c. 对比 #EXT-X-KEY 标签
       └─ 若加密 m3u8 的 KEY 被误删，播放会失败

    d. 检查是否移除了 #EXT-X-DISCONTINUITY 之前的必要片段
       └─ 某些播放器对 DISCONTINUITY 严格

    e. 在 NoAdParser 中调小 ad_keyword_threshold（如 1）
       └─ 或扩大 whitelist_keywords

问题 2：广告片段未被识别

  排查步骤：
    a. 检查 config/noad.php 的 ad_keywords 是否包含广告 URL 中的特征
    b. 调小 ad_keyword_threshold（从 2 改为 1）
    c. 在后台 M3U8解析 Tab 中做在线测试，看命中情况
    d. 编写专用自定义算法（algorithms/*.php）

问题 3：误删正片片段（过度清洗）

  排查步骤：
    a. 增大 ad_keyword_threshold（从 2 改为 3 或更高）
    b. 将正片常用关键词（如正片名称、CDN 域名）加入 whitelist_keywords
    c. 检查是否在自定义算法中误判

问题 4：清洗后的 m3u8 ts 片段跨域失败

  解决：
    启用 noad_proxy.php?mode=ts 作为 ts 代理：
      - NoAdParser 返回 url 时重写片段 URI
      - 或在前端播放器侧配置 proxy 规则

问题 5：m3u8 文件过大，解析超时

  解决：
    - 增大 request_timeout（config/noad.php）
    - 使用代理池分散请求
    - 对超大 m3u8（> 5000 片段）做分片处理（编写自定义算法）
