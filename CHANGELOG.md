### 🚀 性能优化

- **多进程 + curl_multi 双层并发架构**
  - 第一层：pcntl 多进程并行下载，自动检测 CPU 核心数（最高 16 进程）
  - 第二层：每个子进程内使用 curl_multi 并发下载多个 TS 片段
  - 比单进程串行模式提升 5-10 倍下载速度
  - 比单纯多进程模式再提升 2-3 倍

- **MD5 深度分析速度大幅提升**
  - 增大批次：标准模式每批从 10 段提升到 30 段（3 倍）
  - 新增启发式预检测：URL 强特征 + 时长严重异常的片段免下载直接判定，节省 30%-60% 下载时间
  - 新增智能抽样模式：片头 + 片尾 + 中间均匀抽样，仅分析约 25% 片段，速度提升 4 倍

- **默认配置全面升级**
  - 默认并发数从 6 提升到 12（curl_multi）
  - 默认进程数从 4 提升到 8（pcntl 多进程）
  - 自动检测 CPU 核心数上限从 8 提升到 16
  - 智能计算子进程并发数，充分利用带宽

### ⚡ 缓存与性能优化

- **智能缓存机制**
  - 30 分钟内相同 URL 直接返回缓存，秒开播放
  - 缓存包含完整分析结果和纯净 M3U8
  - 支持 `force=1` 强制重新分析
  - 缓存文件锁机制，防止并发重复分析

- **异步后台分析模式**
  - `async=1` 参数：快速返回旧缓存，后台静默更新
  - 用户无感知，始终秒开播放
  - 分析中返回 202 状态码，前端可轮询等待

### 📋 API 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `url` | string | 必填 | 要分析的 M3U8 地址 |
| `mode` | string | `redirect` | 输出模式：`redirect` 跳转 / `json` 返回 JSON |
| `fast` | int | `1` | 快速模式：`1` 启用智能抽样 / `0` 全量分析 |
| `force` | int | `0` | 强制重新分析：`1` 忽略缓存 / `0` 使用缓存 |
| `procs` | int | `0` | 多进程数：`0` 自动检测 / `1-32` 指定进程数 |
| `concurrency` | int | `0` | curl 并发数：`0` 默认 / `1-32` 指定 |
| `sync` | int | `1` | 同步特征码：`1` 自动同步到指纹库 / `0` 不同步 |
| `async` | int | `0` | 异步模式：`1` 秒开 + 后台更新 / `0` 同步等待 |
| `callback` | string | 可选 | JSONP 回调函数名 |

### 📊 返回字段（mode=json）

| 字段 | 说明 |
|------|------|
| `original_url` | 原始 M3U8 地址 |
| `final_url` | 最终解析后的 M3U8 地址 |
| `clean_url` | 去广告后的 M3U8 链接（http/https 开头） |
| `total_segments` | 总片段数 |
| `ad_segments` | 广告片段数 |
| `clean_segments` | 正片片段数 |
| `ad_ratio` | 广告占比（%） |
| `sync_result` | 特征码同步结果（新增数、已存在数） |
| `fingerprint_stats` | 指纹库统计（总数、广告数、白名单数） |
| `ad_indexes` | 广告片段索引数组 |
| `commercial_breaks` | 广告插播段列表 |
| `ad_signatures` | 广告特征码列表（MD5 + 大小 + 时长） |
| `deep_analysis` | 完整深度分析结果 |
| `cached_at` / `cache_expires_in` | 缓存时间信息 |
