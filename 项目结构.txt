【核心数据流向】

  1. 用户请求 → index.php / q.php / noad_md5.php
     └─ 参数校验（URL 非空/格式/协议）
        └─ M3U8 直链检测 → 是 → 直接返回 JSON（M3U8-Direct-Output）
        └─ 否 → 进入对应处理流程

  2. Strategy 决策引擎（index.php 解析模式）
     └─ 匹配平台规则 → 获取优先接口
        └─ 聚合所有可用 API（api.php + ZJK.txt）
           └─ 传给 Requester

  3. Requester 并发请求引擎
     └─ curl_multi 并发请求所有 API
        └─ 收集 HTTP 200 + 非空响应
           └─ Strategy 选最优（平台优先 → 总接口最快 → 全局最快）
              └─ 代理池轮换 + 速率限制（noad.php 配置）
                 └─ 统一响应格式（JSON）

  4. NoAd / MD5 去广告模式（q.php / noad_md5.php / mode=noad）
     └─ NoAdParser
        └─ 资源站匹配清洗
           └─ M3U8 片段广告关键词过滤
              └─ 自定义算法应用（algorithms/ 如意/MD5/模式特征/序列跳跃）
                 └─ SmartConcurrency 双层并发下载 TS 片段
                    └─ MD5 指纹提取 + 特征码自学习（MD5FingerprintDB）
                       └─ SQLite 记录统计

  5. 工具模块（tools/）
     └─ ToolManager 统一管理
        └─ 广告清理 / 批量解析 / 特征提取 / 规则更新 等工具独立调用
